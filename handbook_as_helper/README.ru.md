# handbook_as_helper

[English](README.md) | [中文](README.zh-CN.md) | **Русский**

Используйте сгенерированный **handbook**, чтобы помочь код-агенту. Этот модуль делает ровно
две вещи:

1. **Handbook как планировщик** — превратить handbook в SKILL агента и передать его
   планировщику только для чтения, который локализует *каждое* место правки для запроса на
   изменение на естественном языке (включая разрозненные, неочевидные). **Только план**: он
   выдаёт план; он никогда не редактирует код.
2. **Ресинхронизация handbook** — после того как реальное изменение кода приземлилось,
   прокатить производный слой handbook (карточки, якоря строк, места в коде, индекс) вперёд,
   чтобы он соответствовал изменению, без перегенерации всего handbook.

> Планировщик — это ЕДИНСТВЕННЫЙ агент только для чтения, который маршрутизирует по handbook
> (`SKILL.md` / `index.md` / `registers.md` / `stages/<id>.md`) и сам читает НАСТОЯЩИЙ
> исходный код, прежде чем выдать точный, дословный план ПРАВОК — без под-агента `locator`,
> без map-reduce. Это **единственный** планировщик, и он живёт в `code_agent.py`.
> (Весь код оценки/бенчмарков/выставления оценок — эталонные наборы, A/B-судья, `run_eval.py`,
> фаза исполнителя — был удалён.)

## Установка

```bash
# из корня репозитория
python3 -m venv .venv && source .venv/bin/activate
pip install pyyaml requests tree-sitter tree-sitter-language-pack

# планировщик построен на официальном примере агента NexAU — укажите на этот чекаут
export NEXAU_CODE_AGENT_DIR=/path/to/NexAU/examples/code_agent   # или соседняя директория NexAU/

# LLM: любой OpenAI-совместимый эндпоинт (планировщик И ресинхронизация используют его совместно)
export OPENAI_API_KEY=sk-...                        # обязательно
export OPENAI_MODEL=gpt-4o-mini                     # опционально (по умолчанию: gpt-4o-mini)
export OPENAI_BASE_URL=https://api.openai.com/v1    # опционально; или self-hosted vLLM / прокси
```

Для локального эндпоинта без ключа задайте `OPENAI_API_KEY=EMPTY` (и планировщик, и
ресинхронизация требуют *какой-то* ключ и явно падают, если он не задан). Более низкоуровневые
имена `LLM_MODEL` / `LLM_BASE_URL` / `LLM_API_KEY` по-прежнему учитываются и имеют приоритет
над `OPENAI_*`.

## Часть 1 — handbook как планировщик

### 1. Собрать готовый для планировщика SKILL из handbook

```bash
# универсально (любая цель): собрать handbook_skills/handbook_skill_<target>/ из
# директории отрендеренного handbook (напр. work/repo/handbook или .../phase3/output)
python handbook_skills/build_skill_from_handbook.py --target codex \
    --src /path/to/rendered/handbook

# только Terminus-2: вырезать скилл из полностью отрендеренного markdown handbook
python handbook_skills/build_handbook_skill.py
```

Любая из команд собирает `handbook_skills/handbook_skill_<target>/` = `SKILL.md` + `references/`
(`overview.md`, `index.md`, `registers.md`, `stages/<id>.md`).

### 2. Запустить планировщик для запроса на изменение

```python
import sys; sys.path.insert(0, "pipeline")
from pathlib import Path
from code_agent import run_query             # нужен NexAU + переменные OPENAI_*/LLM_*

out = run_query(
    "<запрос рецензента на изменение на естественном языке>",
    Path("/path/to/source"),        # исходная (pristine) кодовая база для планирования (обязательно)
    Path("runs/case1"),             # временная песочница: git-копия исходников, затем удаляется (обязательно)
    # arm="handbook" по умолчанию (единственный вариант)
)
print(out["plan"])                  # план локализации;  out["diff"] == "" (только план)
```

**Как это работает.** `run_query` делает git-снимок `pristine_dir` в `workdir`, собирает
единственный планировщик только для чтения из официального `code_agent.yaml` от NexAU (с
навигационным handbook, подключённым по пути), прогоняет его по песочнице, затем удаляет
песочницу и возвращает план. Планировщик маршрутизирует по handbook (`SKILL.md` / `index.md` /
`registers.md` / `stages/<id>.md`) и сам читает НАСТОЯЩИЙ исходный код, прежде чем выдать
дословный план ПРАВОК.

## Часть 2 — ресинхронизировать handbook после изменения кода

Ресинхронизация **не зависит от планировщика** (планировщик никогда не редактирует код,
поэтому не создаёт диффа для ресинхронизации). Вы предоставляете готовую *директорию случая*,
описывающую реальное изменение:

```
<case_dir>/
├── edited/       изменённое дерево исходников (напр. чекаут с применённым PR)   [обязательно]
├── plan.md       описание изменения; его объявления управляют согласованием      [обязательно]
└── agent.diff    (опционально) дифф edited/ относительно исходного — пустой дифф пропускается
```

```bash
# на уровне членов (по умолчанию): прокатить производный слой handbook вперёд к изменению
python pipeline/update_handbook.py <case_dir>
python pipeline/update_handbook.py <case_dir> --no-translate   # пропустить LLM-шаг перевода карточек
python pipeline/update_handbook.py <case_dir> --target codex   # выбрать целевой проект

# движок на уровне файлов, для скилла большого конвейера
HANDBOOK_GEN_SCALE=large python pipeline/update_handbook.py <case_dir>
```

Ресинхронизация **многоязычна**: она читает диапазоны / синтаксический шлюз / отпечаток
переименований / граф вызовов из `lang_layer` — Python через `ast`, Rust / TypeScript / Go / …
через tree-sitter-адаптеры из `handbook_generate_small`. Она отказывается только от языка, для
которого не зарегистрирован адаптер. Для цели должно существовать сопоставление фазы 2 на уровне
функций (см. `PHASE2_FINAL` ниже).

## Файлы

| Файл | Назначение |
|------|-----------|
| `pipeline/targets.py` | **Слой конфигурации целевых проектов.** Каждый проект (`terminus2`, `codex`, …) — это `Target`: путь к исходникам, язык, игнорируемое при снимке, формулировки промптов. Добавьте проект здесь — другие файлы менять не нужно. |
| `pipeline/code_agent.py` | **Планировщик handbook** (ветка `handbook`) плюс связующий код: загружает и подставляет переменные окружения в официальный `code_agent.yaml` от NexAU, собирает копию handbook только для навигации (`_ensure_nosrc_handbook`), планировщик только для чтения (`_build` / `build_planner`), одноразовую git-песочницу (`_snapshot_git` / `_git_diff`, также используется ресинхронизацией) и терпимый раннер агента (`_run_agent`). Мостит `OPENAI_*` → `LLM_*` при импорте. Точка входа: `run_query(query, pristine_dir, workdir, arm="handbook")`. |
| `pipeline/update_handbook.py` | **Точка входа ресинхронизации.** Принимает директорию случая (`edited/` + `plan.md`) и прокатывает handbook вперёд к ней (без повторного запуска агента). Выбирает уровень членов или файлов по `HANDBOOK_GEN_SCALE`. |
| `pipeline/resync_handbook.py` | **Движок ресинхронизации на уровне членов** (A→D: семантическая прокатка → вердикт sha → переклассификация → запись в handbook). |
| `pipeline/resync_large.py` | **Движок ресинхронизации на уровне файлов** для скилла большого конвейера (листья — целые файлы; `HANDBOOK_GEN_SCALE=large`). |
| `pipeline/resync_llm.py` | Общий LLM-бэкенд для движков ресинхронизации (`EnvLLM`: один голый POST `/chat/completions` на вызов к тому же OpenAI-совместимому эндпоинту, который используют агенты). |
| `pipeline/resync_decl.py` | Парсер объявлений без членов (используется путём на уровне файлов, чтобы никогда не импортировать движок членов). |
| `pipeline/lang_layer.py` | Многоязычный субстрат: диапазоны, синтаксический шлюз, отпечаток переименований, граф вызовов (Python через `ast`, остальные через tree-sitter-адаптеры). |
| `pipeline/_recon_terminus_base.py` | Помощник Terminus-2: пересобрать каноническое сопоставление `PHASE2_FINAL` / skeleton YAML из отрендеренного handbook. |
| `handbook_skills/build_skill_from_handbook.py` | Универсальный сборщик скиллов: собирает `handbook_skill_<target>/` из любой директории отрендеренного handbook, так что один промпт планировщика работает для разных целей. |
| `handbook_skills/build_handbook_skill.py` | Сборщик скиллов, специфичный для Terminus-2: вырезает скилл с карточками функций из отрендеренного markdown handbook, с обогащением регистрами. |
| `prompts/planner_handbook.md` | Промпт планировщика (маршрутизирует по handbook, читает НАСТОЯЩИЙ исходный код, выдаёт самодостаточные дословные блоки ПРАВОК). |
| `rerun_resync.py` | Нишевый помощник: перезапустить ресинхронизацию для завершённого случая, повторно проиграв его журнал (аблация «с переводом vs без перевода»). Не часть обычного потока. |

## Ключевые переменные окружения

| Переменная | Используется | Значение |
|-----|---------|---------|
| `OPENAI_API_KEY` / `OPENAI_MODEL` / `OPENAI_BASE_URL` | планировщик, ресинхронизация | OpenAI (или OpenAI-совместимый) эндпоинт. По умолчанию: `gpt-4o-mini` / `https://api.openai.com/v1`. Используйте `OPENAI_API_KEY=EMPTY` для локального эндпоинта без ключа. |
| `LLM_MODEL` / `LLM_BASE_URL` / `LLM_API_KEY` | планировщик, ресинхронизация | низкоуровневые переопределения для того же эндпоинта (имеют приоритет над `OPENAI_*`). |
| `NEXAU_CODE_AGENT_DIR` | планировщик | путь к `examples/code_agent` от NexAU (по умолчанию: соседний чекаут `NexAU/`). |
| `EVAL_TARGET` | все | активный целевой проект (`terminus2` по умолчанию, `codex`, …). `--target` переопределяет. |
| `PRISTINE_ROOT` | все | переопределить путь к исходникам цели. |
| `HANDBOOK_SKILL_DIR` / `HANDBOOK_RENDERED_DIR` | сборка скилла / планировщик | переопределить директорию собранного скилла / источник отрендеренного handbook цели. |
| `NEXAU_TOOL_CALL_MODE` | планировщик | формат вызова инструментов (`xml` / `structured`; по умолчанию = `structured` из yaml). |
| `LLM_TEMPERATURE` / `LLM_MAX_TOKENS` / `LLM_MAX_CONTEXT` / `LLM_MAX_ITERATIONS` / `TOOL_OUTPUT_LIMIT` | планировщик | ручки настройки NexAU (по умолчанию: `0.0` / из yaml / `200000` / `300` / `300000`). |
| `LLM_EXTRA_BODY` | планировщик, ресинхронизация | сырой JSON, вливаемый в тело запроса (через `extra_body` из SDK). |
| `HANDBOOK_GEN_SCALE` | ресинхронизация | `large` → движок на уровне файлов; `small` → движок на уровне членов (по умолчанию). |
| `HANDBOOK_GEN_ROOT` | ресинхронизация | явный путь к генератору фаз 2/3, чьи модули запускает ресинхронизация. |
| `PHASE2_FINAL` | ресинхронизация | директория сопоставления фазы 2 / skeleton на уровне функций, которую читает ресинхронизация членов. |
| `HANDBOOK_REFS` | ресинхронизация | переопределить директорию `references/` handbook, которую редактирует ресинхронизация членов. |
| `HANDBOOK_LARGE_SKILL` | ресинхронизация | переопределить директорию скилла большого конвейера, которую редактирует ресинхронизация на уровне файлов. |
| `RESYNC_TRANSLATE` | ресинхронизация | `0` / `off` пропускает LLM-шаг перевода карточек (то же, что `--no-translate`). |
| `RESYNC_NARRATE_LANG` | ресинхронизация | язык текста для ресинхронизации на уровне файлов (large) (`en` / `zh`; по умолчанию `zh`). |

> **Заметка о структуре.** Ресинхронизация на уровне членов по умолчанию читает свои исходные
> артефакты фаз 2/3 из соседнего чекаута `Harness_Translation/` (`PHASE2_FINAL`,
> `PRISTINE_HANDBOOK_JSON`, `UPSTREAM_CACHE`). Задайте `PHASE2_FINAL` (и, при необходимости,
> `HANDBOOK_REFS`), чтобы указать на вашу собственную структуру; пути к JSON фазы 3 / кэшу —
> это опциональное обогащение и пропускаются, если отсутствуют.
